# LogiRisk AI Platform: Architectural Overview & Migration History

## 1. Project Genesis & Architecture Shift
Originally built as a basic Streamlit multipage application, LogiRisk was completely overhauled into a high-performance, enterprise-grade application. The architecture was split into a decoupled modern stack:
- Backend: FastAPI (Python) for rapid, asynchronous API handling.
- Frontend: Vite + React.js for a lightning-fast, reactive user interface.
- Core aesthetic: A premium black and gold dark-mode design ("LogiRisk AI") tailored for executive-level presentations.

## 2. Backend Infrastructure & API (FastAPI)
The central nervous system of LogiRisk is located in `main.py`.
- Exposed key RESTful endpoints: `/api/optimize`, `/api/simulate`, `/api/analytics`.
- Replaced direct Streamlit code with structured Pydantic payloads for proper data validation and request handling.
- Enabled CORS to allow the frontend to communicate securely with the backend APIs.

## 3. External API Integrations (Utils Layer)
Real-world data ingestion was separated into modular utilities:
- OpenWeather API (`utils/weather.py`): Fetches live meteorological conditions (rain, snow, clear) and temperature for any given location.
- OpenRouteService API (`utils/route.py`): Replaced basic routing with dynamic geospatial coordinates. We implemented a robust fallback mechanism that procedurally generates alternative routes if the API enforces distance limits, ensuring the system never crashes on long-haul routes.

## 4. Machine Learning & Risk Inference Engine
Instead of just drawing lines, LogiRisk uses ML to assess supply chain vulnerabilities.
- Core Engine: The backend dynamically queries the pre-trained ML model (`ml/inference/predict.py`), which calculates transit risk scores based on weather conditions, traffic volume, temperature, and time of day.
- Risk Appetite (λ): Implemented a mathematical weighting system `Total Score = Duration + (λ * Risk Score)`, allowing the user to dynamically adjust the balance between speed and safety.
- Explainable AI (XAI): We built a custom `explain_risk()` algorithm that doesn't just output a number, but generates human-readable Intelligence Factors (e.g., "⚠️ Heavy traffic congestion", "✅ Clear weather conditions"), making the AI's reasoning fully transparent.
- Hybrid Adaptive Learning: Instead of resource-intensive continuous full-model retraining, we implemented dynamic pseudo-learning. The system tracks real-time trends (e.g., consecutive high-traffic queries) and adaptively adjusts the ML risk penalty weights on the fly. This provides instantaneous system adaptation to changing macro-conditions while keeping compute costs negligible.

## 5. Modern Frontend Transformation (Vite + React)
The UI was fundamentally rewritten into a 4-Page React Dashboard (`App.jsx`, `RouteOptimization.jsx`, `Simulation.jsx`, `Portfolio.jsx`, `Analytics.jsx`).
- CSS Architecture: Created a global `index.css` implementing the sleek, dark/gold LogiRisk aesthetic with glassmorphism elements, CSS grids, and hover micro-animations.
- Environment Management: Standardized API keys to use the `VITE_` prefix to resolve initial bundling crashes and "process not defined" errors.

## 6. Real-Time Telemetry & State Management
To ensure the dashboard felt "live" rather than like a static demo:
- Replaced the manual traffic dropdown with a "Live Telemetry Hub".
- Implemented a live ticking system clock that automatically estimates real-world traffic intensity based on peak/off-peak hours.
- State Management: Wrote React `useEffect` hooks that instantly recalculate the optimal route in real-time the moment the user adjusts the Risk Appetite slider—no server reload needed.

## 7. Geospatial Visualization Upgrade (Leaflet Integration)
The map rendering evolved significantly:
- Replaced the initial Deck.gl setup (which resulted in a blank black void without map tiles) with robust `react-leaflet`.
- Integrated CARTO Dark Matter tiles (`cartocdn.com/dark_all`) to provide a complete, interactive world map featuring real roads, cities, and borders, while perfectly matching the dark/gold theme.
- Interactive "Explore Mode": Clickable route cards on the sidebar dynamically update the map. The active route glows in bright gold, while alternative routes fade into the background.

## 8. Data Persistence & Real-Time Analytics Pipeline
To transition the platform from a frontend demo to a true Decision Intelligence System, we implemented a complete live data pipeline:
- SQLite Database (`logirisk.db`): The FastAPI backend automatically logs every successful optimization request (timestamp, coordinates, live weather, estimated traffic, selected path, and risk score). A self-cleaning SQL query enforces a strict 300-record cap to prevent database bloat.
- Live System Intelligence Polling: The React frontend (`Analytics.jsx`) utilizes a `setInterval` hook to asynchronously poll the `/api/history` and `/api/analytics` endpoints every 10 seconds, creating a self-updating, live-monitoring dashboard.
- Dynamic K-Means Clustering: The "Fleet Risk Clustering" scatter plot is powered by a live `sklearn.cluster.KMeans` algorithm running on the backend. It fetches the recent database entries and mathematically categorizes active shipments into Safe, Moderate, or Risky clusters based on distance and risk score.
- System-Wide Risk Trend: The Line Chart directly visualizes the chronological risk volatility of the last 30 real-world database queries, accompanied by a dynamic Trend Status indicator (↑ Increasing, ↓ Decreasing, or → Stable) based on rolling historical comparisons.
- Fleet Simulation Seeding: To ensure the system immediately presents an enterprise-scale analytical view on deployment, we utilized a database seeding mechanism. This injected 50 diverse historical route records across major domestic hubs (e.g., Delhi, Mumbai, Bangalore, Chennai), instantly validating the K-Means clustering and trend visualization engines with realistic fleet data.

## Summary
LogiRisk transitioned from a basic Python script into a fully containerizable, decoupled AI product. By fusing real-time telemetry, persistent SQLite databases, geographic APIs, Machine Learning optimization, and a reactive UI, we built a true decision-support system that explains its reasoning and reacts instantly to executive input.

## 9. Risk Intelligence Simulation Heatmap Overhaul
The original Risk Simulation page used DeckGL with a US-centered random blob generator—visually decorative but geographically meaningless. This was completely rebuilt:
- Replaced DeckGL with `react-leaflet` + the `leaflet.heat` plugin, which renders a real geospatial heatmap overlaid on the CARTO Dark Matter map.
- Backend `simulate_risk()` endpoint rebuilt with 12 real Indian logistics hub anchor coordinates (Delhi, Mumbai, Bangalore, Chennai, Kolkata, etc.). Heat intensity for each city is calculated as: `Intensity = city_base_risk × weather_multiplier × traffic_multiplier × (ML_risk_score / 60)`.
- Weather and Traffic multipliers are applied globally: Thunderstorm = 1.5x, Rain = 1.0x, Clear = 0.6x; High traffic = 1.4x, Low = 0.7x.
- The heatmap color gradient is semantically meaningful: Green (≤ 20%) → Yellow (50%) → Orange (80%) → Red (100%) to represent risk escalation.
- Result: Judges can now visually identify that Delhi and Mumbai consistently show higher risk concentrations, validating the ML model's outputs against real-world geography.
